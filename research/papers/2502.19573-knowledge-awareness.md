# Do Large Language Models Know How Much They Know?

> Source: arXiv:2502.19573v3 (26 Feb 2025, full HTML fetched 2026-06-29). Prato, Huang, Parthasarathi, Sodhani, Chandar (Chandar Lab / Mila / Meta FAIR / Polytechnique). EMNLP 2024. PDF in pdf/.

## Thesis

A capable system should know the scope of its own knowledge. Tested directly: can a model enumerate everything it knows about a topic — no more, no less? Awareness emerges, but only with sufficient scale; below it, models recall a random count (too few, or hallucinate extra). This locates the knowledge boundary empirically.

## Method

Fine-tune on synthetic diary entries; each fictitious diarist authors 1–8 entries of 1–8 attributes. Prompt: "Recall all of {name}'s diary entries, in order." Exact-match accuracy on count + content. Synthetic to avoid pretraining contamination; 6 datasets, 1K→64K diarists. Suites: OPT (7M–2.7B), Pythia (70M–2.8B), Flan-T5 (80M–3B).

## Results

| Finding | Number |
| --- | --- |
| Capability emerges with scale | yes, all suites; rate differs by architecture |
| OPT-125M vs Pythia-160M, large data | OPT far ahead → emergence is architecture-paced |
| Pre-trained vs scratch (32K), Pythia-70M | 21.56% → 45.50% (pretrained weights hurt small) |
| OPT-125M pretrained vs scratch | 91.94% vs 82.19% |
| Document length / count effect | none up to 8 docs (capacity not yet saturated) |

## Use in book
