# LLM-as-Code: Agentic Programming for Agent Harness

> Source: arXiv:2606.15874v1 (14 Jun 2026, full HTML). Qi et al., City Univ. Hong Kong.

## Thesis

Frameworks make the LLM the orchestrator; token explosion, control-flow hallucination, unreliable completion are architectural, not bugs — deterministic work (loop/branch/sequence) given to a probabilistic system. Fix: program owns control flow; LLM = adaptive component (LLM-as-Code) called only for reasoning.

## Four parts

- Code-driven workflow: runtime enforces rules, not begged from model. Compliance not sampled.
- DAG context: history = call tree; live context = ancestor chain, returned children collapse to one-line summary → O(depth) not O(steps).
- Multi-agent = parallel function calls, typed returns. Self-programmed evolution: propose f' (LLM), accept only if tests pass, commit as code.

## Evidence

OSWorld GUI: 86.8% in 15 steps vs best baseline 80.4% in 100 steps. Top-level loop ~150 LOC. vs LangGraph (pre-declared graph), DSPy (tunes calls not who owns flow), CodeAct (code as action not loop).

## Use

Ch3/Ch4: deterministic harness owns loop; bounded context by depth; verify-by-construction.
