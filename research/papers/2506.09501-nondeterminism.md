# 2506.09501 — Understanding and Mitigating Numerical Sources of Nondeterminism in LLM Inference

> Source: <https://arxiv.org/abs/2506.09501> — Jiayi Yuan, Hao Li, Xinheng Ding, Wenya Xie, Yu-Jhe Li, Wentian Zhao, Kun Wan, Jing Shi, Xia Hu, Zirui Liu (11 Jun 2025). PDF in `pdf/2506.09501.pdf`.
> Companion (engineering): Horace He & Thinking Machines Lab, "Defeating Nondeterminism in LLM Inference", Sep 2025 — <https://thinkingmachines.ai/blog/defeating-nondeterminism-in-llm-inference/>

## Thesis

LLM benchmark scores are not reproducible. Changing the **evaluation batch size, GPU count, or GPU version** changes the generated responses, because floating-point arithmetic on GPUs is non-associative — `(a+b)+c ≠ a+(b+c)` under finite precision — so the reduction order, which depends on system configuration, changes the numbers. The effect is worst in **reasoning models**: a minor rounding difference in an early token can cascade into a divergent chain of thought and a different final answer. The paper characterises the sources and proposes mitigations (e.g. higher-precision accumulation) that reduce but do not fully remove the variance.

He / Thinking Machines refine the root cause: the LLM forward pass is actually *run-to-run* deterministic, but its kernels are not **batch-invariant**. Because a shared inference server's load — and therefore its batch size — varies from request to request, each user's output depends on what else happened to be in the batch. Demonstration: Qwen3-235B, 1000 completions of one prompt at **temperature 0**, produced **80 unique completions** (identical for the first 102 tokens, then "Queens, New York" vs "New York City"). Batch-invariant kernels restore true determinism at a modest performance cost.

## Use in book

- **Ch1 Capabilities & Limitations** (added): even at temperature 0 a model is not reproducible; you cannot treat it like deterministic software, so any test or check must allow for variation. Cited as [He, 2025] + [Yuan et al., 2025].
- **Ch1 Key Takeaways** (added): "It predicts; it does not know" now notes non-reproducibility.
- **Ch3 / Ch4** (potential): reinforces observability/trajectory analysis over deterministic unit testing for agent loops; reproducibility is a system property, not a model setting. Pairs with the harness "correctness gate" argument.
