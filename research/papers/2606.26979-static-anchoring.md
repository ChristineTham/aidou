# How Much Static Structure Do Code Agents Need? Deterministic Anchoring

> Source: arXiv:2606.26979v1 (25 Jun 2026, full HTML). Lin, Zhou, Yang, Li (Beihang). github.com/mathieu0905/Code-Anchor. CodeAnchor on Codex/GPT-5.1; SWE-bench Lite (274) + Verified (500).

## Thesis

Grep-first agents miss structure (call graphs, inheritance, config deps) → stochastic, irreproducible navigation. Inject offline static analysis (PyCG + AST) as inline plain-text tags = "deterministic anchors". Helps less by making smarter, more by making navigation disciplined+reproducible. Passive injection > MCP tool (pilot: agents ignored tool, used grep). Codex baseline beats graph LocAgent by 23.7pp.

## Three findings

- **Works:** Anchor-Topo +2.2pp Func@5, −1.6 rounds; localization→repair +12.5pp on differential subset (strict superset). **Scale-sensitive:** dense saturates (+18.8% tokens, no gain); hub-heavy repos favor inverse-only (who-calls-me). **Stabilizes:** LFR 0.15→0.21-0.24, halves variance, Pass@1 +3.4pp; ~10% more tokens. Variants: Topo/Dense/Inv; heuristic out-degree<5 Topo, >8 Inv.

## Use

Ch4 context engineering: structural anchors, reproducibility as first-class; medium repos benefit most.
