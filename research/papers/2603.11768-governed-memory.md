# 2603.11768 — Governing Evolving Memory in LLM Agents (SSGM Framework)

> Source: <https://arxiv.org/abs/2603.11768> — Chingkwun Lam (12 Mar 2026). PDF in `pdf/2603.11768.pdf`.

## Thesis

Once an agent's memory is something it writes to and edits over time, the memory itself becomes a risk surface. The paper catalogues failure modes of *evolving* memory — drift, corruption, contradiction, and leakage of private data — and proposes the **Stability and Safety Governed Memory (SSGM)** framework: a governance layer that decides what may be written, when stale or conflicting entries are evicted, and what must be verified before anything enters the long-term store. Memory management is treated as a control problem with explicit policies (and optionally learned ones), separating the agent solving the task from the mechanism curating its memory.
