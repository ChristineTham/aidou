# Runtime-Structured Task Decomposition for Agentic Coding Systems

> Source: arXiv:2605.15425v1 (14 May 2026, full HTML). Asthana et al., IBM Research/Zoom. ACM CAIS '26. CC BY 4.0. Impl: Mellea.

## Thesis

Monolithic prompts = brittle, costly full reruns. RSTD: task partition by executable control flow, LLMs only for schema-validated judgment, failures isolated → selective retry. 3 components: decomposition engine, typed judgment operators, state manager.

## Results (10 runs, gpt-4 t=0)

- K8s RCA: static retry cost 1,632±145 > monolithic 904±17 (+80.5% — cascading downstream rerun); RSTD 436±132 = 51.7% under mono, 73.2% under static. Debugging: 933 vs 703 vs 460. Framework overhead ~18%.
- Key: runtime branching (not decomposition structure) is the recovery mechanism. RSTD baseline higher (multi-call) — worth it when failure rates non-trivial.

## Use

Ch3 coding patterns: subtask-level retry, per-subtask model substitution, AgentOps monitoring.
