# NanoKnow: How to Know What Your Language Model Knows

> Source: arXiv:2602.20122v2 (30 Apr 2026, full HTML fetched 2026-06-29). Gu, Jedidi, Lin (University of Waterloo). SIGIR 2026, DOI 10.1145/3805712.3808604. PDF in pdf/.

## Thesis

What a model knows is largely a function of pretraining frequency. Using nanochat (open FineWeb-Edu data), project NQ/SQuAD onto the corpus into "supported"/"unsupported" splits to disentangle parametric vs external knowledge — making the knowledge boundary measurable, not guessed.

## Method

BM25 retrieve top-100 → answer string match → Qwen3-8B verify. 97.2M docs, 326GB index. 66.2% NQ / 70.9% SQuAD supported. Eight nanochat checkpoints, d20 (561M), d32 (1.9B), d34 (2.2B); EM + Qwen3-14B judge; closed-book, FineWeb context, distractors (Far/Mid/Near).

## Results

| Finding | Number |
| --- | --- |
| Closed-book accuracy vs answer frequency (Rare→High) | >2× increase; nil at d20 (no capacity) |
| Closed-book NQ, d20 → d34 | +26.9 pts (19.2%) |
| Unsupported closed-book EM (boundary) | 0.8% NQ / 1.5% SQuAD |
| Distractor-only vs closed-book | −3.2 / −1.5 pts → noise hurts |
| Answer + 1 → 4 distractors (NQ, Far) | 0.447 → 0.378 |
| Lost-in-the-middle | least accurate with answer between distractors |
