# The Spec Growth Engine: Spec-Anchored, Code-Coupled, Drift-Enforced

> Source: arXiv:2606.27045v1 (25 Jun 2026, full HTML). Hartwig Grabowski, Hochschule Offenburg.

## Thesis

Two AI-coding failure modes: (1) context explosion ("Dumb Zone" past ~40% fill; context rot); (2) silent spec-code drift. Lean middle ground: spec-anchored + code-coupled (same commit) + drift-enforced (blocking merge). Synthesises Parnas, C4, ADRs, Walking Skeleton, Reflexion Models, fitness functions — no RUP/MDA overhead.

## Five mechanisms

- Spec graph: node = one SPEC.md, C4 levels, contract(outward)/design(inward) split; ownership tree + dependency DAG (contract only, one hop).
- Spine: ownership path root→N + dep contracts + own code; excludes siblings/dep designs/transitive — scoped context = Parnas hiding.
- Vertical slices hardest-first: Layer 1 invariants up front, Layer 2 features just-in-time; frontier stubs off active path only.
- Drift gate: Intent graph (specs) vs Evidence graph (code); orphan/undeclared-dep block merge; AI updates spec in same commit.
- Governance gates HARD/SOFT/AUTO by blast-radius. Capability registry = DRY at arch level.
